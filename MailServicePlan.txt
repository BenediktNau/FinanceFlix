Plan: Mail Inbox Integration with Local AI Categorization

 Context

 FinanceFlix needs to:
 1. Associate one or more IMAP mail inboxes (with a specific folder) per Account
 2. Listen for new incoming emails in real-time using IMAP IDLE
 3. When a new email arrives, use a local AI (Ollama) — no internet required — to:
   - Extract the transaction amount from the bill/receipt
   - Determine the TransactionCategory (Einkommen, Wohnen, Lebensmittel, etc.)
 4. Automatically create a Transaction record linked to the Account

 ---
 Prerequisites (one-time setup)

 Install Ollama locally: https://ollama.com
 Then pull the recommended lightweight model:
 ollama pull phi3:mini
 phi3:mini (Microsoft Phi-3, ~2.2 GB RAM) is the best choice for structured JSON extraction — fast, efficient, good at following instructions and handles German text well. No internet needed after the initial
  pull.

 Alternatives if RAM is limited: llama3.2:1b (~1.3 GB) or gemma2:2b (~1.6 GB).

 ---
 Architecture Overview

 New email arrives (IMAP IDLE)
        ↓
 MailListenerService (BackgroundService)
        ↓
 OllamaCategorizationService → POST http://localhost:11434
        ↓ returns { category, amount }
 CreateTransactionCommand → ITransactionRepository → DB

 File: FinanceFlix/Services/AI/OllamaCategorizationService.cs

 Uses OllamaSharp to call the local Ollama API:

 public class OllamaCategorizationService : ICategorizationService
 {
     private readonly OllamaApiClient _ollama;

     public OllamaCategorizationService(IConfiguration config)
     {
         var host = config["Ollama:Host"] ?? "http://localhost:11434";
         var model = config["Ollama:Model"] ?? "llama3.2";
         _ollama = new OllamaApiClient(host) { SelectedModel = model };
     }

     public async Task<(TransactionCategory Category, decimal Amount)> CategorizeAsync(string subject, string body, CancellationToken ct = default)
     {
         var prompt = $"""
             You are a financial transaction categorizer for a German finance app.
             Given this bill/receipt email, extract:
             1. The category (must be exactly one of: Einkommen, Wohnen, Lebensmittel, Transport, Unterhaltung, Gesundheit, Shopping, Sparen, Sonstiges)
             2. The total amount as a decimal number (e.g. 29.99)

             Email Subject: {subject}
             Email Body: {body}

             Respond ONLY in JSON: {{"category": "...", "amount": 0.00}}
             """;

         var response = await _ollama.GetCompletion(prompt, ct);
         // Parse JSON response → map to TransactionCategory enum and decimal
         // Use System.Text.Json to deserialize
         var result = JsonSerializer.Deserialize<CategorizationResult>(response.Response);
         var category = Enum.Parse<TransactionCategory>(result.Category);
         return (category, result.Amount);
     }
 }

 Step 6 — IMAP IDLE Background Listener

 File: FinanceFlix/Services/Mail/MailListenerService.cs

 BackgroundService that:
 1. On startup: loads all MailInbox records from DB (via IServiceScopeFactory)
 2. Spawns one Task per inbox, each running an IMAP IDLE loop on inbox.FolderName
 3. On new message: fetch it → call ICategorizationService → dispatch CreateTransactionCommand
 4. Re-issues IDLE every 25 min (servers drop connections at ~30 min)
 5. Retries with 30s delay on connection failure

 private async Task OnNewMessageAsync(MailInbox inbox, IMailFolder folder, int messageIndex, CancellationToken ct)
 {
     var message = await folder.GetMessageAsync(messageIndex, ct);

     using var scope = _scopeFactory.CreateScope();
     var categorizer = scope.ServiceProvider.GetRequiredService<ICategorizationService>();
     var mediator = scope.ServiceProvider.GetRequiredService<IMediator>();

     var (category, amount) = await categorizer.CategorizeAsync(message.Subject, message.TextBody ?? "", ct);

     await mediator.Send(new CreateTransactionCommand(amount, message.Subject, category, inbox.AccountId), ct);
 }
